# Sistema de pedidos de feature

## Entrada

- Canal único (ex.: formulário, email `support@`, ou GitHub Discussions quando ativo).

## Avaliação

| Pergunta | Peso |
|----------|------|
| Alinha com visão? | Alto |
| Impacto estimado | Alto |
| Custo eng + risco | Médio |
| Urgência legal | Crítico |

## Resposta ao utilizador

- Template: “Obrigado — registámos como `IDEA-###`. Estado: em análise / planeado / não planeável por [razão].”

## Ligação com roadmap

- Ideias aceites entram em `roadmap.md` (canónico) após priorização.
