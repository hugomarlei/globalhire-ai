# Política de reembolso (interna)

> **Público:** alinhar com páginas legais do site. Este doc é **operacional interno**.

## Princípios

- Clareza no período de reembolso e condições (Stripe Customer Portal quando activo).
- Registar todos os reembolsos com motivo (qualidade, erro billing, fraude).

## Fluxo interno

1. Verificar subscrição no Stripe (customer id)
2. Aplicar reembolso parcial/total conforme política
3. Notificar utilizador com template
4. Marcar em CRM/ticket
