# Política de reembolso (suporte)

> Alinhar texto com páginas legais. Este documento é **guia operacional** para agentes de suporte.

## Antes de reembolsar

- [ ] Verificar elegibilidade temporal (janela definida comercialmente)
- [ ] Confirmar identidade da conta (email verificado)
- [ ] Verificar se houve uso abusivo (fraude)

## Execução

- Stripe Dashboard → reembolso
- Registar motivo: `QUALITY`, `BILLING_ERROR`, `FRAUD`, `GOODWILL`

## Pós-reembolso

- Confirmar email ao utilizador
- Se churn por qualidade — abrir item em `PRODUTO/bug-triage.md` ou feedback
